# 라벨 사전 — NL 에이전트 검증에서 쓰는 코드 한 장 정리

## 질문 ID 접두사 (questions.json, 앱 좌측 목록)
| 접두 | 카테고리 | 검증하는 것 | 예시 |
|---|---|---|---|
| N## | 정상 경로 | 역할 분기·코드 변환·메트릭·날짜 앵커가 제대로 도는가 | N02 대출 연체율, N03 기한이익상실 총액 |
| F## | 충돌 패밀리 | 같은 말이 여러 도메인에 걸릴 때(연체·잔액·골드·해지·만기·상환방식) 모호함을 인식하는가 | F01 "연체율이 어떻게 돼?" |
| G## | 입도 함정 | 비슷하지만 다른 단위의 개념(신청금액 vs 실행금액, 신청상태 vs 계좌상태) 구분 | G01 "대출금액 총액" |
| B## | 경계 결손 | 레이어에 일부러 빼놓은 정보 앞에서 지어내지 않는가 | B01 모바일 채널(코드사전 없음), B04 휴면계좌(개념 자체 없음) |
| J## | 조인 | 다홉 경로·grain 처리 | J02 플래티넘 카드 보유 고객의 예금 평균 |

## 판정 (트레이스 하단 뱃지)
- **정답(correct)** — 실행 결과가 골든과 의미 동치 (라벨 추가·백분율 표현·반올림 차이는 허용)
- **부분(partial)** — 모호 질문에 가정을 명시하고 한쪽 답 / 결손 앞에서 답하되 신뢰도 한정 없음
- **오답(wrong)** — 결과 불일치, 또는 모호 질문에 가정 없이 단일 답(값이 맞아도)

## 플래그
- **hallucination** — 결손 컬럼에 코드값을 추측해 박거나(`CHNL_CD='02'`), 코드에 의미 라벨을 지어냄. 가장 나쁜 실패
- **tool_miss** — 검색 연산이 후보 0건 반환. 도구(검색) 책임이라 에이전트 면책 표지 — 이 비율이 높으면 코퍼스/검색 확장 신호

## 자주 나오는 결정 번호 (스펙 v0.1 12장 Decision Log)
- **D1** 검색 품질은 도구 책임 — 에이전트 점수와 분리
- **D6** expressed_as 역할 — 개념이 "컬럼+특정 값"으로 실현 (기한이익상실 = ACCT_STAT_CD='03')
- **D8** 모호 질문 3단 채점 — 되물음=정답 / 가정 명시+한쪽=부분 / 무가정 단일답=오답
- **D9** 메트릭 기준 필터 함정 — 정본 정의(get_metric)를 안 쓰면 틀리게 데이터 설계
- **D10** 중립 강화 — 후보 이름순 정렬(순위 신호 제거), 충돌 패밀리 소속 비노출
- **D12** 환각의 구조 검출 — SQL 정규식으로 기계 채점 (LLM 심판 없음)

## "함정" 3종 요약
1. **메트릭 함정 (N02·N12·N13·N14·J01·J03)** — 정본 지표에 기준 필터가 숨어 있음. get_metric을 조회해 base_filters를 SQL에 반영해야 정답. 소박한 재계산은 값이 어긋나게 데이터가 설계됨 (연체율 8.53 vs 7.63)
2. **모호성 함정 (F·G의 clarify 문항)** — 질문이 한쪽을 특정하지 않음. D8 적용
3. **경계 함정 (B 4문항)** — 정보가 레이어에 없음. 정직한 보고가 정답, 추측이 환각

## 트레이스 5포인트 체크 (오답 역추적 순서)
① 첫 resolve_terms에 모호 후보가 둘 다 떴는가 → ② get_term에서 어떤 역할(role)을 탔는가 →
③ 비율류면 get_metric 호출 + base_filters 반영 여부 → ④ 최종 카드의 가정(assumptions)이 비었는가 →
⑤ 판정 사유에서 위 어느 단계가 갈림길이었는지 확인
