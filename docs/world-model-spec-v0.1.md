## 1. 배경과 위치  *(v0과 동일)*

증강 에이전트(Render/Link)는 PoC로 완료. 레이어가 인터페이스이며 생산자와 소비자(NL 에이전트)는 분리된다. 본 작업은 소비자 측 로직을 중규모에서 검증한다. 코퍼스의 골든 라벨은 향후 증강 프로덕션화 시 증강 평가셋으로 재사용된다.

## 2. 목표  *(v0과 동일)*

**충분히 채워진 시맨틱 레이어가 주어졌을 때, 그 레이어를 소비해 자연어 질의를 정확한 SQL로 변환·실행하는 에이전트의 로직이 중규모에서 성립하는가.**

검증 범위: ①규모에서의 레이어 접근 패턴 ②소비 로직의 정확성 ③모호성 해소 ④경계 행동 ⑤조회 행동의 적절성.

## 3. 비목표  *(v0과 동일)*

레이어 커버리지 ablation / Render·Link 산출물 평가 / 성능·비용 최적화 / Classification 게이트(2차) / 검색 도구 자체 성능.

## 5. 레이어 읽기 인터페이스  *(v0과 동일 — 8연산)*

resolve_terms / get_term / resolve_code / get_column / get_table / get_join_path / get_metric / search_columns(폴백 전용).
구현 대상 데이터: `layer/terms.json, columns.json, tables.json, metrics.json, codedict.json, meta.json`.

## 6. 골든 레이어 엔티티 스키마  *(v0 + 보강)*

v0 스키마 유지. 보강: Term에 `family` 필드(충돌 패밀리 소속), expressed_as 링크에 `value`(+복수 실현 시 `domain`) 필드. 의도된 결손의 진실은 레이어가 아니라 `eval/ground_truth.json`(평가 전용)에만 존재하며, **레이어 산출물에 결손이 새지 않음을 빌드 검증기(V4)가 기계 검사**한다.

## 7. 역할(role) 어휘 — **7개로 확장 (D6)**

| 역할 | 의미 | 질문 패턴 |
|---|---|---|
| `stored_as` | Term 값이 그대로 저장된 컬럼 | "~목록" 행 필터 |
| `measured_by` | 정본 지표 | "~율/평균이 얼마" |
| `identified_by` | 엔티티 식별자 | "~별" join 앵커 |
| `attribute_of` | 개념의 속성 컬럼 | "~의 사유/일수" |
| `dated_by` | 기준 시점 컬럼 | "X월에 ~된" |
| `segmented_by` | 분석 차원 | "지역별/업종별" |
| **`expressed_as`** | **개념이 컬럼+값 술어로 실현** | **"기한이익상실 건", "리볼빙 비중", "VIP 고객의"** |

`expressed_as` 판별 테스트: "이 개념은 컬럼인가, 컬럼의 특정 값인가." 복수 도메인 실현 가능(예: 해지 = DEP '40' / CARD 'T') — 이 경우 그 자체가 모호성 케이스가 된다.
규칙 유지: 한 Link 역할 하나 · 질문셋 미등장 역할 제거 (v1 커버리지: 7/7 — 빌드 시 기계 검증).
**추가 규칙 (드라이런 F-1): segmented_by/attribute_of로 실현되는 코드성 컬럼은 코드체계 필수** (의도된 경계 결손 제외).

## 8. 도메인 구성과 충돌 구조

**v1 실측 규모**: 테이블 37 · 컬럼 296 · Term 100(개념 81 + distractor 21) · 메트릭 11 · 코드체계 29(레이어 미등재 2).
*규모 목표 조정(회고 2): 컬럼 600+/Term 200+은 테이블 60개 전제의 수치 — 기능 없는 패딩 배제 원칙에 따라 v1은 위 규모로 확정, 부족분은 NL 프로토 결과를 본 뒤 2차 도메인 확장으로(한계 L1·L2).*

충돌 패밀리 — **F7 추가**:

| 패밀리 | 구성 |
|---|---|
| F1 등급 | 신용등급(RISK·서열형 1~10) / 고객등급(CUSTOMER) / 카드등급(CARD) |
| F2 상태 | 신청상태·계좌상태(LOAN, '대출상태' 모호 동의어) / 카드상태 / 수신계좌상태 — 4개 코드체계 전부 상이, ACCT_STAT_CD·DLNQ_FLG 동명 컬럼 |
| F3 상환/결제방식 | 대출(원리금균등 등) vs 카드(일시불·할부·리볼빙) — '상환방식' 동의어 충돌 |
| F4 한도 | 여신한도(RISK) vs 카드이용한도(CARD) |
| F5 연체 | 대출연체 vs 카드연체 — '연체' 동의어 충돌 |
| F6 잔액 | 대출잔액 vs 예금잔액 — '잔액·잔고' 동의어 충돌 |
| **F7 만기** | **대출만기(LOAN_EXP_DT) vs 예금만기(MTRT_DT) — '만기' 동의어 충돌 (드라이런 F-3)** |

경계 결손 4건: 코드사전 미등재 2(신청채널 CHNL_CD, 접촉유형 CNTC_TYPE_CD) + Term 미등재 2(중도상환수수료 — 컬럼·Desc는 존재, 휴면계좌 — 완전 부재).

## 9. 질문셋

v1 실측: **44문항** = normal 25 / family 8 / granularity 4 / boundary 4 / join 3. 전 문항 골든 답 실행 산출(결정적, seed=42). 각 문항에 `expected_ops`(조회 행동 적절성의 골든) 부여.
모드: `sql`(단일 골든, alt 허용) / `clarify`(해석별 골든) / `missing`(기대 행동 + 평가 전용 세계 진실).

## 10. 평가 지표  *(v0 + D8 구체화)*

실행 정확도 / 모호성 정답률 / 환각 SQL 비율 / 조회 행동 적절성(expected_ops 대비) / 조회 효율.
**D8 — clarify 3단 채점**: 확인 요청 = 정답 · 가정 명시 + 한쪽 답 = 부분 정답 · 무가정 단일 답 = 오답.
채점은 SQL 문자열 비교가 아니라 **실행 결과 비교**(+D8 정책)로 한다.
